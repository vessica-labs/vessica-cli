database, so the default plugin restoration entrypoint is unavailable.

## Initial diagnosis

1. Explicit trigger decisions work in both positive and negative controls.
2. `ves prime` is broken for hosted attachments.
3. Stored knowledge currently contains workflow history, not durable product
   decisions, facts, or guidance.
4. Context assembly admits unrelated active artifacts before relevant memories,
   allowing artifact volume to crowd out retrieval.
5. Empty lexical results plus weak query guidance encourage expensive synonym
   fan-out by the plugin.
6. The repository-scoped CLI is sufficient for the feature track but does not
   yet provide an obvious person/calendar/CRM scope for a chief-of-staff model.

The first improvement series should keep trigger behavior fixed while repairing
hosted prime, empty-result handling/query guidance, and artifact admission. Only
then should ranking weights or embeddings be compared.
